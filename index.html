What is Semantic HTML?

Semantic HTML refers to the practice of using HTML tags that convey the meaning and structure of the content they contain, rather than just its appearance. For example, using <h1> for the main heading, <nav> for navigation links, and <article> for a blog post tells both browsers and assistive technologies what each part of the page is for.

This contrasts with non-semantic tags like <div> or <span>, which provide no context about their content.

Why Accessibility Matters

A web built with semantic markup is inherently more accessible. Users who rely on assistive technologies, like screen readers, can navigate the page more effectively.

  • Landmarks: Tags like <header>, <main>, and <footer> create "landmarks" that allow users to jump directly to key sections.
  • Clear Hierarchy: A logical heading structure (<h1>, <h2>, <h3>...) creates a navigable outline of the page.
  • Context: A screen reader will announce "navigation" for a <nav> element, giving the user immediate context.

SEO and Semantic Tags

Search engine crawlers (like Googlebot) are sophisticated users that also benefit from clear structure. By understanding the content hierarchy and meaning, they can index your site more accurately.

A clear structure helps search engines identify the most important content on your page, which can positively influence your search ranking. The page title, main heading, and meta description are all critical signals.

Did You Know?

Search engines often use the content of the <meta name="description"> tag to generate the snippet shown below your page title in search results.
